# MoneyIQ

## Screens:

1. Home Screen - Shows live stock news in a 3 column layout with stock prices of top 10 stocks somewhere in between
2. Login Screen - accepts username and password
3. Signup Screen - accepts name, username, email id and password
4. KYC Screen - accepts aadhaar card number, phone number
5. User Dashboard - shows live stock, ipo and mutual fund prices in a floating navbar with 3 tabs
6. Admin Dashboard - has a floating navbar with 2 tabs, the first tab allows to upload new expert videos (from either youtube or from device), the second tab allows to add todays expert advice(s) and also shows history of all the previous advices
7. User Profile - has a sidebar with 2 tabs, the first tab has the user details - name, username, password, email id, phone number - and has options to update each one too (after some kind of verification via email or phone number). the second tab has the user's portfolio in a pie chart layout showing the composition of each bought stock. There are some metrics below the pie chart as well

## API Details

This section describes the API for the stock and IPO prices data using the AngelOne SmartAPI
This involves hitting two APIs:

1. `https://margincalculator.angelbroking.com/OpenAPI_File/files/OpenAPIScripMaster.json`
   This API gives you the symbols supported by AngelOne, needs to be updated just once per day hence it has less overhead
   The response format is and array of such objects:

```json
{
	"token": "99926000",
	"symbol": "Nifty 50",
	"name": "NIFTY",
	"expiry": "",
	"strike": "0.000000",
	"lotsize": "1",
	"instrumenttype": "AMXIDX",
	"exch_seg": "NSE",
	"tick_size": "0.000000"
}
```

This would be happening in the nextjs server from which it updates the UI
The UI then sends the request to the websocket for the prices 2. `https://apiconnect.angelone.in/rest/secure/angelbroking/market/v1/quote/`
Getting these prices is then trivial via this second API
The sample request is:

```NodeJS
  var axios = require('axios');
  var data = JSON.stringify({
      "mode": "FULL",
      "exchangeTokens": {
          "NSE": ["3045"]
      }
  });

  var config = {
  method: 'post',
  url: 'https://apiconnect.angelone.in/
  rest/secure/angelbroking/market/v1/quote/',
  headers: {
      'X-PrivateKey': 'API_KEY',
      'Accept': 'application/json, application/json',
      'X-SourceID': 'WEB, WEB',
      'X-ClientLocalIP': 'CLIENT_LOCAL_IP',
      'X-ClientPublicIP': 'CLIENT_PUBLIC_IP',
      'X-MACAddress': 'MAC_ADDRESS',
      'X-UserType': 'USER',
      'Authorization': 'Bearer AUTHORIZATION_TOKEN',
      'Content-Type': 'application/json'
  },
  data : data
  };

  axios(config)
  .then(function (response) {
  console.log(JSON.stringify(response.data));
  })
  .catch(function (error) {
  console.log(error);
  });
```

The response is:

```json
{
	"status": true,
	"message": "SUCCESS",
	"errorcode": "",
	"data": {
		"fetched": [
			{
				"exchange": "NSE",
				"tradingSymbol": "SBIN-EQ",
				"symbolToken": "3045",
				"ltp": 571.75
			}
		],
		"unfetched": []
	}
}
```

Maximum requests of this API - per second: 10, per minute: 500, per hour: 5000

Usage of the AngelOne websocket is refrained since it involves a lot of complexity, poor documentation and is still under development

The websocket server calls the API once every 1 second and caches it preventing direct access of the API from client side which could easily bypass the rate limits of the API
